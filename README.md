## 🔐 IOTA — Invisible Ownership & Traceability Architecture

<div align="center">

<table width="700" cellpadding="0" cellspacing="1">
<tr>
  <td bgcolor="#1C6BA4" align="left">
    <font color="white"><b>&nbsp;&nbsp;📋&nbsp; Project Overview</b></font>
  </td>
  <td bgcolor="#1C6BA4" align="right">
    <font color="#88BBDD"><b>─ &nbsp;□ &nbsp;✕&nbsp;&nbsp;</b></font>
  </td>
</tr>
<tr><td colspan="2" bgcolor="#EEF4FB">
<table width="100%" cellpadding="14" cellspacing="0">
<tr><td>

**IOTA** is a purpose-built invisible watermarking system for **artist copyright protection**. It embeds survivable ownership metadata directly into image pixels — imperceptible to the human eye, fully recoverable by any verifier.

Unlike existing systems that only track AI provenance, IOTA answers a harder question:

> *"Who owns this content — and has it been tampered with?"*

<br/>

| Layer | Coded Bits | Embeds | Survives |
|:---:|:---:|---|---|
| **L1** | 255-bit BCH | Owner identity · registry pointer | 70% crop · JPEG q50 · platform re-encoding |
| **L2** | 127-bit BCH | License · metadata pointer | Standard social media pipelines |
| **L3** | Fragile seal | Tamper evidence | Nothing — breaks on any modification |

<br/>

**Architecture highlights:**
- 🧠 U-Net encoder (~31M params) with content-coupled keying and saliency masking
- 🔑 Per-image key derivation via `HMAC-SHA256(global_key, SHA-256(image))`
- 🛡️ Public verification API hardened against gradient-estimation oracle attacks
- 🗂️ C2PA-compliant copyright registry with perceptual collision detection
- 🔍 CLIP/SimCLR fingerprint database fallback for AI-regenerated content
